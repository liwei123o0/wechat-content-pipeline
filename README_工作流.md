# 微信公众号 AI 科技新闻发布工作流

## 📋 目录结构
```
wechat_publisher/
├── run_all.py            # 一键运行：采集->选题->深度搜索
├── publish_latest.py     # 一键发布：读取最新文章->发布到草稿箱
├── step1a_采集_AI科技.py
├── step2_选题_AI科技.py
├── step3_深度搜索.py
├── publish_v3.py         # 底层发布 API
├── 发布历史_去重.py
├── 封面生成器.py
├── 整理/                 # 选题结果
├── 素材/                 # 深度搜索素材
├── 创作/                 # 生成的文章 (.md)
└── output/               # 发布副本
```

## 🚀 使用方式

### 方式 1：Cron 自动运行（推荐）
```
⏰ 每 12 小时自动运行一次（00:00 和 12:00）
📤 完成后结果自动发送到微信对话
```

### 方式 2：手动触发完整流程
```bash
# 1. 准备数据（采集 + 选题 + 深度搜索）
cd /home/lw/wechat_publisher && python run_all.py

# 2. 在 Hermes 对话中生成文章（Agent 自动完成）
# 3. 发布到草稿箱
python publish_latest.py
```

### 方式 3：只发布已有的文章
```bash
# 如果文章已经生成好了，直接运行：
python publish_latest.py
```

| 功能 | 说明 |

## 🖼️ 封面图策略（新增）

1. **优先使用新闻原图**（最真实，最匹配）
   - 从新闻页面的 `og:image` / `twitter:image` 提取
   - 自动裁剪为微信标准比例：1280×544（2.35:1）
   - 支持 JPG/PNG/WebP 等格式

2. **AI 生成作为备份**
   - 如果新闻没有配图，自动调用 AI 生成
   - 科技风格，匹配文章关键词
   - 已内置于 `publish_v3.py`
**修复前：** `step4_创作.py` 依赖 MiniMax API + 环境变量，cron 环境缺少配置  
**修复后：** 由 Hermes Agent 直接用当前 LLM 模型（豆包）生成文章 ✅

### 问题 2：Step 5 发布脚本失效
**修复前：** `step5_发布.py` 用 wenyan CLI，路径不对且经常静默失败

### 问题 3：Cron 任务流程复杂
